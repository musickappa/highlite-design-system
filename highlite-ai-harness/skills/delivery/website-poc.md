# Skill: Website PoC — フェーズ1（営業時）

## 目的
営業フェーズで見込みクライアントへ提示するPoC（概念実証）Webサイトを、ヒアリング情報を基に短時間で生成する。

---

## フロー

### STEP 1 — クライアント情報ヒアリング

以下の項目をユーザーに確認する（わかっている項目はスキップ可）。

**[A] 会社基本情報**
1. 会社名（日本語 / 英語）
2. 業種・業態
3. タグライン / キャッチコピー（あれば）
4. 会社概要（2〜3文で）

**[B] サイト目的**
5. サイトを作る一番の目的は？（例：リード獲得 / ブランド認知 / 採用 / EC）
6. ターゲット読者は誰か？
7. 訪問者に伝えたい核心メッセージは？

**[C] ブランド**
8. メインカラー（HEXがあれば）
9. フォントのイメージ（モダン / 伝統的 / カジュアル など）

**[D] コンテンツ**
10. ヒーローのキャッチコピー（見出し + サブ見出し）
11. 提供サービス or 機能（2〜4項目）
12. 数字で語れる実績・規模感（例：創業10年 / 顧客数500社）
13. CTAボタンの文言とリンク先
14. お問い合わせ先メールアドレス

---

### STEP 2 — profile.json の作成

ヒアリング結果を基に `clients/_template/profile.json` をコピーして
`clients/<client-slug>/profile.json` を作成・入力する。

`client_slug` の命名規則: 小文字英数字 + ハイフン（例: `acme-corp`, `yamada-foods`）

---

### STEP 3 — PoC生成

```bash
node website-system/generate-poc.js <client-slug>
```

出力先: `clients/<client-slug>/poc/index.html`

---

### STEP 4 — 確認・調整

生成後に以下を確認する：
- [ ] ブランドカラーが意図通りに反映されているか
- [ ] コピーに誤字・違和感がないか
- [ ] サービスカード数が適切か（推奨: 3〜4枚）
- [ ] CTAリンクが設定されているか

調整が必要な場合は `profile.json` を直接編集して再生成する。

---

## ファイル構成

```
clients/
  <client-slug>/
    profile.json          ← 入力データ
    poc/
      index.html          ← 生成されたPoC
```

## 引き渡し先

PoC完成後、案件化した場合は `website-build.md`（フェーズ2）へ移行する。
`_meta.phase` を `"poc"` → `"active"` に更新し、`delivery/SKILL.md` の案件登録フローを実行する。

---

## 注意事項

- PoC段階では `poc-template.html` のセクション構成を変更しない（変更はフェーズ2で）
- クライアント名・ロゴは正式許可前は `poc_notes.sales_context` に記録する
- 生成されたHTMLはそのままクライアントへメール添付または画面共有で見せられる状態にする
